How to use this hostel management system

This page explains the full setup in a normal, step-by-step way. If you are opening this project for the first time, follow the steps in order and you will have the app working with your own Supabase project.

Before you start

You only need a few things before beginning.

  • A Supabase account.
  • This project folder on your computer.
  • One email address and password that you want to use as the hostel admin login.

Simple idea

This project is just a frontend. Supabase is used for login, database tables, and image storage. So the setup is mostly about creating a Supabase project and connecting this app to it.

Step 1

Create a Supabase account

Go to Supabase and sign up if you do not already have an account. After logging in, you will reach the Supabase dashboard.

Step 2

Create a new project

Click the option to create a new project. Give it a name, choose a database password, and select the region you prefer. Then wait until the project finishes creating.

Step 3

Create your admin login

Inside Supabase, open Authentication, then Users, and create a user manually. Enter the email and password that you want to use for this hostel system login page.

Step 4

Open the SQL editor

In your Supabase project, open the SQL Editor. This is where you will create all the tables that this app needs.

Step 5

Run the database setup SQL

Copy the SQL from the SQL queries to run section below, paste everything into the Supabase SQL Editor, and run it. This creates the required tables, relationships, policies, indexes, and storage buckets.

Step 6

Set up storage buckets

This project uses storage for student photos and hostel event images. If the SQL creates them for you, that part is already handled. If not, create the buckets that the project expects, such as student-photos and hostel-events.

Step 7

Copy your project URL and anon key

In Supabase, open Project Settings and then the API section. Copy the project URL and the public anon key. These two values are needed to connect this app to your Supabase project.

Step 8

Update supabase-config.js

Open supabase-config.js in this project. Replace the placeholder project URL and placeholder anon key with your real Supabase values. Do not change the rest of the file unless you know why you are changing it.

Step 9

Open the project in your browser

Start or open the project the same way you normally do on your computer, then go to the login page. If the configuration is correct, the login button should work instead of showing a setup warning.

Step 10

Log in with the admin user

Use the email and password you created in Supabase Authentication. After login, you should reach the dashboard.

Step 11

Start adding your real hostel data

Now you can begin using the system normally. Add rooms first, then add students, then use attendance, installments, notifications, complaints, and events as needed.

Step 12

Change branding if you want

If this system is for another hostel or college, replace the logo image, background image, and any visible college name text so the project matches your own institution.

If something does not work

  • If login is disabled, check whether supabase-config.js still has placeholder values.
  • If data is not loading, make sure the SQL setup finished successfully in Supabase.
  • If uploads fail, check that storage buckets exist and storage access is allowed.
  • If your login fails, confirm the user was created in Supabase Authentication and the password is correct.

Recommended order after setup

  • Create rooms.
  • Add students and assign rooms.
  • Use attendance to track present and absent students.
  • Use installments and transactions for fee tracking.
  • Use notifications, complaints, and events for daily hostel management.
